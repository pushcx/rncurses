ncurses-ruby - A Ruby module for accessing the ncurses library

About ncurses

ncurses is a C library with fuctions for manipulating text screens. Thus, it is useful for programmers who write interactive applications that have to run in text terminals. ncurses is abailable here as a GNU software package. Ncurses provides functions for input and output from and to text terminals (providing random access to any screen coordinate), and embeds these functions into a framework of rectangular regions on the screen.

If you want to learn more about ncurses, then you can either read

  • the ncurses-intro text that is included in the ncurses distribution (a copy of ot is available here),
  • a few sections from the ancient Linux Programmer's Guide, starting here, or
  • the new ncurses programming howto
You will need a basic understanding of ncurses if you want to use the ncurses-ruby module, since it is a wrapper of the underlying C library.

About ncurses-ruby

This ruby extension makes most functions, constants, and external variables of the C library ncurses accessible from the Ruby programming language.

All C functions are wrapped by module functions of a the module "Ncurses", with exactly the same name. Additionally, C functions expecting a WINDOW* as their first argument can also be called as methods of the "Ncurses::WINDOW" class.

The panel library (for support of overlapping windows) is also wrapped, in the module "Ncurses::Panel".

Using ncurses-ruby on a windows system

ncurses works well in a cygwin environment on windows. If your version of the ruby interpreter was compiled with cygwin, you can easily use the ncurses library provided by the cygwin project to compile ncurses-ruby.

If you'd rather use a non-cygwin version of ruby, you may be able to get ncurses-ruby running together with the PDCurses library. The ncurses-ruby source package contains a description how to compile ncurses-ruby for a MinGW-compiled ruby interpreter.

In the Download area, there is also a precompiled ncurses-ruby Package to use with a MinGW-compiled ruby.

Going further

This BerliOS project aims at more than wrapping ncurses. The goal is to provide a Textmode User Interface Application Framework on the basis of the ncurses wrapper. Ncurses enables applications to manipulate text terminals, but it still requires a lot of work from the programmer to build user interfaces from that.

Such a framework would provide User interface Widgets like push buttons, checkboxes, menubars etc, and also an event-driven callback framework. Imagine something like Borland's "Turbo Vision" framework implemented in ruby.

To learn more about the requirements of what such a framework, I plan to implement a ruby wrapper for turbo vision.
